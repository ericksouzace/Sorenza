# Sorenza Design System — Quiet Intelligence

## Design thesis

Sorenza should feel technologically advanced because of what it understands and reveals, not because it glows.

The visual language is deliberately quiet:

- warm neutral canvas;
- ink-first typography;
- restrained semantic color;
- soft borders instead of excessive cards;
- serif display typography for editorial confidence;
- compact sans-serif for operational precision;
- motion only where it communicates state.

## Product references

The product takes principles—not visual copies—from modern AI interfaces:

- conversation/task first;
- generous whitespace;
- progressive disclosure;
- clear state transitions;
- minimal chrome;
- direct manipulation;
- immediate feedback.

## Interaction rules

1. One dominant action per surface.
2. Destructive/irreversible actions always explain impact.
3. Autonomy is a policy, not a decorative toggle.
4. Progress is stage-based and evidence-based; never fake percentage loading.
5. Every error says what failed and what the user can do next.
6. Complex controls appear only when context requires them.
7. Status color is never the only status signal.
8. `prefers-reduced-motion` is respected.
9. Keyboard focus is always visible.
10. Small operational metadata remains secondary but readable.

## Main information architecture

- Início — compose and observe the current mission.
- Projetos — connected local/Codespaces environments.
- Missões — durable verified history.
- Equipe — agent responsibilities and benchmark status.
- Skills — versioned capabilities.
- Evidências — independent proof and promotion state.

Market/lead discovery belongs to a later `Labs`/business module and must not compete with the engineering mission flow.
